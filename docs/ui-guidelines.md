# UI 작업 가이드라인

프로젝트: 복슬달리기(BokslRunning)  
기준 화면: 홈 화면(`app/src/main/java/com/boksl/running/ui/feature/home/HomeScreen.kt`)

## 1. 문서 목적

- 홈 화면에서 정리한 시각 톤을 다른 화면에도 일관되게 적용한다.
- 페이지마다 따로 만든 느낌이 아니라 하나의 제품처럼 보이게 한다.
- MVP 범위 안에서 과한 장식보다 정보 전달과 사용 흐름을 우선한다.

## 2. 현재 UI 방향

- 전체 톤: 블랙 기반 다크 UI
- 인상: 차분하고 밀도 있는 피트니스 앱
- 구조: 검은 배경 위에 둥근 차콜 카드가 떠 있는 형태
- 강조 방식: 한 화면에 강한 포인트 색은 최소 개수만 사용
- 우선순위: 큰 숫자 -> 보조 레이블 -> 액션 버튼 순서로 읽히게 설계

## 3. 컬러 규칙

홈 화면 기준 기본 팔레트:

- 배경: `#000000`
- 카드 배경: `#1B1B1E`
- 보조 버튼/아이콘 영역: `#38383C`
- 기본 텍스트: `#F5F5F5`
- 보조 텍스트: `#A2A2A7`
- 포인트 그린: `#8ED548`
- 포인트 블루: `#66A1FF`
- 포인트 오렌지: `#FF8A5B`

사용 원칙:

- 화면 배경은 기본적으로 검정 유지
- 주요 정보는 차콜 카드 위에 배치
- 한 카드 안에서 포인트 색은 최대 2~3개까지만 사용
- CTA는 그린 우선
- 에러/주의가 아니면 빨간색 남용 금지

## 4. 여백 규칙

- 화면 좌우 기본 여백: `16dp`
- 상단 영역 시작 여백: `18dp` 내외
- 카드 내부 여백: `24dp`
- 카드 간 세로 간격: `22dp`
- 같은 카드 안의 정보 블록 간 간격: `18dp ~ 20dp`
- 작은 텍스트와 숫자 사이 간격: `8dp`

원칙:

- 여백은 넉넉하게 두되, 의미 없는 빈 공간처럼 보이면 안 된다.
- 카드 내부 정렬축은 되도록 좌측 기준으로 맞춘다.
- 상단 아이콘이나 보조 버튼은 화면 끝에 바짝 붙이지 않는다.

## 5. 라운드 규칙

- 큰 카드 radius: `30dp`
- 메인 버튼 radius: `20dp`
- 보조 버튼 radius: `18dp`
- 작은 점/배지/원형 처리: `999dp`

원칙:

- 카드와 버튼 모두 둥근 형태를 유지하되, radius를 제각각 쓰지 않는다.
- 같은 위계의 컴포넌트는 같은 radius를 유지한다.

## 6. 타이포 규칙

- 화면 타이틀: 강한 볼드
- 핵심 수치: `headlineLarge` 또는 `titleLarge` 급
- 보조 정보: `bodyMedium`, `bodySmall`
- 보조 라벨은 연한 회색, 핵심 숫자는 흰색

원칙:

- 의미 없는 섹션 제목은 만들지 않는다.
- 숫자가 핵심인 화면에서는 숫자가 가장 먼저 읽혀야 한다.
- 타이틀 아래 설명 문장은 꼭 필요할 때만 둔다.

## 7. 텍스트/카피 규칙

- 불필요한 장식 텍스트 금지: 예) `러닝`, `운동`, `READY` 같은 의미 약한 라벨
- 상태를 쓰려면 실제 의미가 있어야 한다: 예) `위치 권한 필요`, `프로필 필요`
- 버튼 문구는 행동 중심으로 쓴다: 예) `달리기 준비`
- 영문/한글 혼용은 최소화한다.

표기 규칙:

- 달린 시간: `000시간 00분`
- 칼로리: 천 단위 콤마 사용
- 거리/속도 단위는 화면 전체 톤에 맞춰 일관되게 유지

## 8. 카드 구성 규칙

홈 화면 기준 추천 구조:

1. 카드 상단: 가장 중요한 숫자 또는 액션
2. 카드 본문: 보조 지표 2~3개
3. 카드 하단: 행동 버튼 또는 부가 설명

원칙:

- 장식용 그래프/바는 금지
- 그래프나 바를 쓰려면 목표값, 비교값, 시간축처럼 해석 기준이 있어야 한다
- 정보가 없으면 단순 텍스트 레이아웃이 더 낫다

## 9. 버튼 규칙

- 메인 CTA는 화면당 1개만 강하게 강조
- 보조 CTA는 중립 회색 버튼으로 처리
- 상단 설정 버튼은 배경 없는 작은 아이콘 우선
- 버튼 레이블은 짧고 분명하게 유지

예시:

- 메인: `달리기 준비`
- 보조: `기록`, `통계`

## 10. 아이콘 규칙

- 설정은 톱니바퀴 아이콘 사용
- 아이콘은 기능 의미가 분명할 때만 사용
- 아이콘 배경 원형은 꼭 필요할 때만 사용
- 아이콘 자체보다 주변 여백 정렬이 더 중요하다

## 11. 다른 화면 적용 원칙

기록 화면:

- 리스트 배경은 검정 유지
- 기록 아이템은 카드형 차콜 표면 사용
- 날짜보다 거리/시간/페이스가 먼저 읽히게 배치

통계 화면:

- 차트가 필요하면 유지하되, 차트 없는 숫자 요약 카드도 함께 둔다
- 차트 색은 그린/블루 중심으로 제한
- 축/가이드라인은 과하게 드러나지 않게 한다

설정 화면:

- 폼 화면이어도 같은 카드 톤 유지
- 설명 문장은 짧게, 섹션 수는 최소화
- 주요 진입 액션만 강조

## 12. 금지사항

- 연한 회색/보라 위주의 임시 느낌 팔레트 재도입 금지
- 의미 없는 상태 텍스트 추가 금지
- 해석 기준 없는 진행 바/장식 바 추가 금지
- 한 화면에 강한 포인트 색을 여러 개 난립시키는 구성 금지
- 여백 규칙 없이 컴포넌트를 화면 끝에 붙이는 배치 금지

## 13. 작업 체크리스트

다른 페이지 작업 전 아래를 확인한다:

- 이 화면도 검정 배경 + 차콜 카드 구조를 유지하는가
- 핵심 숫자/핵심 액션이 가장 먼저 보이는가
- 의미 없는 제목이나 상태 배지가 들어가 있지 않은가
- 버튼 위계가 명확한가
- 포인트 색 수가 과하지 않은가
- 카드 간 간격과 화면 좌우 여백이 홈 화면과 비슷한가
- 텍스트가 한글 중심으로 정리되어 있는가
- 장식이 아니라 정보로 설명 가능한 요소만 남아 있는가

## 14. 추후 정리 방향

- 공통 색상/spacing/radius를 Compose 토큰으로 분리
- `HomePalette` 수준의 하드코딩 값을 앱 공통 디자인 토큰으로 승격
- 공통 카드/버튼/헤더 컴포넌트를 추출해 화면별 편차를 줄임
