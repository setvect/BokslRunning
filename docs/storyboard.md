# 1) 첫 실행 스토리보드 (프로필 + 권한)

### 트리거

* 앱 최초 실행

### 화면/상태

**S1-1 온보딩**

* 메시지: “러닝 경로/시간/페이스를 기록합니다”
* 버튼: `시작하기`

**S1-2 프로필 입력**

* 입력: 몸무게(kg), 성별, 나이
* 버튼: `저장`

**S1-3 위치 권한 요청**

* 메시지: “경로 기록을 위해 위치 권한이 필요해요”
* 버튼: `허용`, `나중에`

### 사용자 행동 → 앱 반응/결과

* `저장` → 위치 권한 요청 화면으로 이동
* `허용` → 권한 승인 후 홈 이동
* `나중에/거부` → 홈 이동, 러닝 시작 시 권한 안내

---

# 2) 권한 거부/미허용 스토리보드

### 트리거

* 위치 권한 미허용 상태에서 `러닝 시작`

### 화면/상태

**S2-1 권한 안내**

* 메시지: “위치 권한이 없으면 경로를 기록할 수 없어요”
* 버튼: `설정 열기`, `취소`

### 사용자 행동 → 앱 반응/결과

* `설정 열기` → OS 설정 진입 유도
* `취소` → 홈 복귀

---

# 3) 홈/설정 스토리보드

### 트리거

* 권한 단계 완료 후 홈 진입

### 화면/상태

**홈**

* 상단: `설정` 버튼
* 누적 요약 카드:
  * 총 달린 거리
  * 달린 시간
  * 평균 속도
  * 소모 칼로리
* 버튼: `러닝 시작`, `기록`, `통계`

**설정**

* 버튼: `프로필 정보 수정`, `전체 내보내기`, `가져오기`, `홈`

### 사용자 행동 → 앱 반응/결과

* `설정` → 설정 화면 이동
* `러닝 시작` → 권한 상태 확인 후 Ready 또는 권한 안내 이동
* `기록` → 기록 목록 이동
* `통계` → 통계 화면 이동

---

# 4) 러닝 시작 전(Ready) 스토리보드

### 트리거

* 홈에서 `러닝 시작`

### 화면/상태

**S3-1 Ready 화면**

* 지도: 현재 위치 점
* 오프라인 배지/안내(오프라인일 때)
* 버튼: `시작`, `취소`

### 사용자 행동 → 앱 반응/결과

* `시작` → 러닝 라이브 진입, 기록 시작
* `취소` → 홈

---

# 5) 러닝 중 스토리보드

### 트리거

* Ready에서 `시작`

### 화면/상태

**S4-1 러닝 라이브 화면**

* 지도 + 경로
* 실시간 지표:
  * 러닝 시간
  * 거리
  * 현재 페이스
  * 평균 페이스
  * 최고 속도
  * 칼로리(추정)
* 오프라인 안내(오프라인일 때)
* 버튼: `종료`

### 사용자 행동 → 앱 반응/결과

* 러닝 중 지표/트랙 포인트 지속 갱신

---

# 6) 종료 확인/저장 스토리보드

### 트리거

* 러닝 화면에서 `종료`

### 화면/상태

**S6-1 종료 확인**

* 메시지: “러닝을 종료하고 저장할까요?”
* 버튼: `저장`, `취소`

**S6-2 결과 요약**

* 카드: 총 거리/총 시간/평균 페이스/최고 속도/칼로리
* 지도: 전체 경로
* 버튼: `완료`

### 사용자 행동 → 앱 반응/결과

* `저장` → 기록 저장 후 요약 화면 표시
* `완료` → 기록 목록 이동

---

### 트리거

* 오프라인 상태에서 러닝 시작

### 화면/상태

### 사용자 행동 → 앱 반응/결과

# 8) 앱 중단 후 이어서 기록 스토리보드

### 트리거

* 러닝 중 앱/프로세스 중단 후 재실행

### 화면/상태

**S8-1 이어서 기록 안내**

* 메시지: “진행 중이던 러닝이 있어요. 마지막 저장 지점부터 이어서 기록할까요?”
* 안내: “앱이 꺼져 있던 구간은 기록되지 않을 수 있어요”
* 버튼: `이어하기`, `폐기`

### 사용자 행동 → 앱 반응/결과

* `이어하기` → 러닝 화면 복귀, 이어서 기록
* `폐기` → 세션 종료 처리 후 기록 목록 이동

### 사용자 행동 → 앱 반응/결과

# 10) 통계 스토리보드

### 트리거

* 홈에서 `통계`

### 화면/상태

**S10-1 통계 화면**

* 전체 누적 요약:
  * 지금까지 총 달린 거리
  * 지금까지 달린 시간
  * 지금까지 평균 속도
* 월별 차트(과거 → 최근):
  * 월별 총 달린 거리
  * 월별 달린 시간
  * 월별 평균 속도
* 초기 진입 시 최신 구간이 먼저 보이고, 좌우 스와이프로 더 과거 월을 탐색할 수 있음
* 차트를 터치하면 선택한 월의 상세 값이 하단 카드에 표시됨

### 사용자 행동 → 앱 반응/결과

* 기록이 추가되면 통계/차트 값 갱신
* 차트를 왼쪽으로 스와이프하면 더 과거 월 데이터가 표시됨

# 11) 전체 데이터 내보내기 스토리보드

### 트리거

* 설정에서 `전체 내보내기`

### 화면/상태

**S11-1 내보내기 화면**

* 버튼: `내보내기 시작`
* 안내: “모든 러닝 기록을 파일로 생성합니다”
* 형식: `bokslrunning_export_v1.json`

**S11-2 진행 상태**

* “N개 중 M개 생성 중…”
* 버튼: `취소`

**S11-3 완료**

* 메시지: “내보내기 완료”
* 버튼: `파일 공유/저장`, `홈`

### 사용자 행동 → 앱 반응/결과

* 내보내기 파일 1개 생성 후 공유/저장 가능

---

# 12) 데이터 가져오기(백업 후 병합) 스토리보드

### 트리거

* 설정에서 `가져오기`

### 화면/상태

**S12-1 가져오기 화면**

* 버튼: `가져오기 시작`
* 안내: “가져오기 전에 현재 데이터를 내부 백업합니다”
* 정책: 기존 데이터 유지 + 가져온 데이터 병합

**S12-2 진행 상태**

* “1/2 내부 백업 중…”
* “2/2 데이터 병합 중…”
* 버튼: `취소`

**S12-3 완료**

* 메시지: “가져오기 완료”
* 결과: “추가 N건, 중복 M건”
* 버튼: `확인`, `홈`
* 동일 파일 재가져오기 시: “이미 가져온 파일입니다” 안내 후 종료

### 사용자 행동 → 앱 반응/결과

* `가져오기 시작` → 내부 백업 후 병합
* 완료 시 결과 안내 후 기록/홈으로 이동
